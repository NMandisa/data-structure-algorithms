FAMILY 1 The O(n²) Trio

Algorithm How It Works Stable? Adaptive? Best Use Memory Hook
🫧 Bubble Compare neighbours, swap if wrong. Biggest "bubbles" to end each pass. ✅ Yes ✅ Yes
O(n)		 Educational only Neighbours swap, heaviest rises.
🎯 Selection Scan unsorted, find min, swap to front. Repeat. ❌ No ❌ No When swaps are expensive Find min, put first. Always n².
🃏 Insertion Pick next card, slide left into sorted prefix. ✅ Yes ✅✅ Strong Nearly sorted / small n Slide left. Hidden QuickSort booster.
⚡ Head-to-Head: Nearly sorted? → Insertion. Costly swaps? → Selection. Else? → Insertion. Bubble is rarely preferred.

FAMILY 2 The O(n log n) Big Three

Algorithm Mechanism Stable? Space Worst Case Why It Matters
🔀 Merge Divide → sort halves → merge sorted fronts ✅ Yes O(n) Θ(n log n) Stable guarantee. External sort friendly.
⚡ Quick Pivot → partition left/right → recurse ❌ No O(log n) stack O(n²) ⚠️
Trigger: sorted + bad pivot		 Fastest in practice. Cache-friendly. Fix: random pivot.
🏗️ Heap Build max-heap from n/2−1 → 0. Swap root↔last, shrink, heapify root. ❌ No O(1) Θ(n log n) Guaranteed O(n log n). Truly in-place. Cache-unfriendly.

FAMILY 3 The Cheaters O(n)

Sort How It Cheats Time Stable? Requirement
Counting Counts → prefix sum → direct index O(n+k) ✅ Yes Integer keys [0, k)
Radix LSD/MSD digit passes + stable sub-sort O(d·(n+k)) ✅ Yes Fixed-width digit keys
Bucket Scatter → sort buckets → concat O(n) avg ⚠️ Depends Uniform distribution
🔑 Pattern: Skip Ω(n log n) by exploiting key structure. Not general-purpose.

📊 Stability Truth Table

✅ STABLE
• Bubble Sort
• Insertion Sort
• Merge Sort
• Counting / Radix

❌ UNSTABLE
• Selection Sort
• Quick Sort
• Heap Sort
• Shell Sort

🔍 Complexity Patterns

• Two separate loops → O(n)
• Nested loops → O(n²)
• Loop halves/doubles → O(log n)
• Outer normal + inner log → O(n log n)
• Constant step i+=3 → Still O(n)

📈 Where to Focus

📝 5 Golden Sentences

  1. QuickSort uses O(log n) stack — NOT O(1)
  2. HeapSort is NOT stable — use Merge for stable O(n log n)
  3. Sequential loops → ADD. Nested → MULTIPLY
  4. Recursive depth ≈ log₂n for divide-and-conquer
  5. Only Merge among O(n log n) sorts is stable
